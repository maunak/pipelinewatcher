    Fabric Pipeline Watcher — Quick Guide

    Getting Started

    • Add Tenant — click + Add Tenant and enter your Azure tenant ID
    • Favorites — click ☆ on a pipeline to pin it; favorites load first on startup
    • Workspace filter — use the workspace picker to focus on one workspace, star or blacklist workspaces

    Dashboard

    • Search — filter by pipeline or workspace name
    • Status pills — filter by Failed, Succeeded, Never run, In Progress (multi-select)
    • Sort — click any column header to sort ascending/descending

    Quick Actions

    Hover a row to reveal action buttons:

    • ↺ Refresh last run   ⬇ Fetch full history   ▶ Re-run pipeline
    • 📋 Copy Run ID   🔗 Open in Fabric   📊 View history panel

    History Panel

    • Chart — duration over time with color-coded statuses
    • Annotations — add notes on specific dates (click 📝 icon)
    • Period — switch between 7d, 30d, 90d, or all

    Alerts & Patterns

    • Failure notifications appear automatically as VS Code alerts for favorited pipelines
    • Configure daily report time in settings (default 18:00)
    • Pattern detection warns if a pipeline fails repeatedly on the same day or time window

    Settings

    Open VS Code settings and search pipelineWatch:

    • pollingInterval — auto-refresh rate in seconds (default 60)
    • retentionDays — days to keep history (default 90)
    • batchSize / batchDelayMs — tune API batch loading
    • blacklistedWorkspaces — exclude workspaces by name or ID
